[TARENA] Run Metagame DB/UI Cleanup Summary

Date: 2026-06-16

Goal: make run-metagame frontend scripts use persisted Offline DB state instead of sample armies, placeholder ids, temporary stores, or automatic dev data writes.

Changed

  • Added OfflineRunContextDbReader as the shared DB reader for active run context, Start Run record reload, screen-specific next_screen lookup, persisted summaries, latest async battle result id, and snapshot conversion.
  • Added OfflineRunContextDbWriter as the shared DB writer for offline_runs creation and run-context updates.
  • Start Run now creates offline_runs through the run context writer and reloads the persisted CreatedRunRecord through the run context reader.
  • Added OfflineDatabaseLegacyIdentity.ToLegacyRouteNodeId so route node facade labels are centralized.
  • Connected OfflineModeDatabaseComposition to create the run context reader and to pass the PRD035 generation catalog into Run Shop DB store.
  • Reward Map now loads a persisted run whose next_screen is Reward, then reads current army and run gold from DB.
  • Run Shop now loads a persisted run whose next_screen is RunShop, then reads current node, current army, and run currency from DB.
  • Summary Value now loads persisted run_summaries instead of building sample start/pre-final/post-final armies.
  • Battle Result now loads the latest persisted async result instead of recording a hardcoded sample result on awake.
  • Saved Armies no longer auto-seeds a slot or writes sample attack history during screen/service initialization.
  • Run Map DB store now rejects missing runs instead of inserting mock-start-army rows.
  • Run Map, Run Battle, Reward Map, Run Shop, Summary Value, and Start Run no longer contain direct INSERT/UPDATE offline_runs SQL outside the shared writer.
  • Run Map mockup was explicitly isolated onto an in-memory mock store so it cannot write fake run records into the production Offline DB.
  • Prefab builders no longer serialize removed sample/placeholder fields.

Removed Runtime Temporary Data

  • No runtime offline-run, offline-run-final-win, offline-shop-node, or offline-async-result-027 placeholders in screen controllers.
  • No hardcoded Reward Map, Run Shop, Summary Value, or Battle Result sample armies in runtime screen controllers.
  • No automatic Saved Armies seed slot or mock history writes.
  • No Run Map DB creation of placeholder offline_runs rows.
  • No duplicated runtime offline_runs writer SQL outside OfflineRunContextDbWriter.

Verification

Unity compilation and EditMode tests were not run automatically. This project requires Unity-side validation.
  • Static scans found no remaining runtime sample army builders or placeholder screen fields in the changed controllers.
  • Run DB store construction remains centralized in OfflineModeDatabaseComposition.
  • INSERT INTO offline_runs and UPDATE offline_runs now scan only inside OfflineRunContextDbWriter.
  • Mockup-only in-memory use is excluded from the production-composition guard.

Remaining Follow-Up

PRD035 seed persistence is still encoded through generated route ids. A future schema migration should add first-class run seed metadata before analytics, replay, or online authority depends on it.
