Run StackRepresentation Prefab Flow

Zasada dla Start Run UI: StartRunScreenController wybiera dwa prefaby. startingArmyPrefab to caly prefab karty armii (StartRunArmyCardView) instancjowany raz na kazda starting army. armyDetailsPrefab to prefab jednego stack-rowa z StackRepresentation, instancjowany raz na kazdy stack wybranej armii.

Flow danych

  1. DB/service zwraca DTO lub snapshot stackow armii.
  2. RunMetagameDisplayInfoFactory mapuje stack na StackInfoData.
  3. RunMetagameStackListPresenter.DisplayStackInfo czysci stare sample rows, instancjuje prefab i znajduje StackRepresentation.
  4. StackRepresentation.DisplayStackInfo renderuje ikone, liczbe, wartosc i dane unitow.

Zmienione ekrany

  • 020_StartRun: StartRunScreenController instancjuje army cards z startingArmyPrefab i stack details z armyDetailsPrefab.
  • 021_RunMap: army panel uzywa DB snapshotu i tego samego helpera prefabowego.

Regula programowania

Nie budujemy produkcyjnych list stackow z child-name lookupow, mock-rowow ani statycznych tablic sample row. Static rows moga istniec tylko jako template prefabow, nie jako zrodlo danych UI.
