Start Run: DataMapper, DB i generator

Graf pokazuje, skąd Start Run bierze dane i jak łączy authored data z DB progress/unlock state.

UI / ekran kod C# / adapter ScriptableObject authored data SQLite DB runtime/progress generator wynik dla UI / run

Najkrótszy opis flow

  1. StartRunScreenController.cs prosi adapter o dane ekranu.
  2. OfflineModeDatabaseComposition.cs składa service i tworzy DataMapperStartRunUnitSource.
  3. DataMapperStartRunUnitSource woła DataMapper.Instance.
  4. DataMapper.Instance ładuje raz Resources/0_Data/DataMapper, czyli ScriptableObject z referencjami do katalogów.
  5. UnitCatalog i SkillCatalog dają authored data: jednostki, koszty, tiery, faction, role i skille.
  6. DB daje runtime/progress data: unlocki konta oraz później zapis wybranego runu i snapshotu.
  7. DeterministicRunGenerationCatalog łączy authored unit pool z DB unlockami i generuje oferty armii.
  8. StartRunService zamienia oferty na UI rows, a po starcie zapisuje wybraną armię do DB przez store/writer.
