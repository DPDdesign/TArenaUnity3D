Co dziala jako kod

  • Da sie wystartowac runa ze starting army.
  • Da sie przejsc do node typu Battle i zagrac bitwe.
  • Widok bitwy dziala, ale UI i swiatlo sa do poprawy.
  • Armia startowa idzie z katalogu Mock_startingArmyRuleSet.
  • Armia przeciwnikow idzie z EnemyArmyGenerator.
  • Encountery ida z Mock_EnemyEncounters.
  • Run Map ma typy: Start, Battle, Shop, RecruitReward, FinalBoss, RandomEvent, Empty.
  • PRD037 materializuje seed, node'y, connections i enemy placeholders.
  • EnemyEncounterRuleCatalog istnieje jako SO dla Low/Medium/High/Boss.
  • Runtime SQL nie zapisuje juz route_maps, route_paths, ani route_nodes.

Czego brakuje

BrakMapa jako SO

Topologia jest nadal w DeterministicRunGenerationCatalog.BuildPaths.

BrakEnemy catalog integration

Katalog trudnosci nie jest jeszcze uzywany przez generator, map_node_enemies ani Run Battle.

BrakPredefined armies catalog

Nie ma jeszcze katalogu, ktory mapuje PredefinedEnemyId na konkretna armie wroga, bossa albo special encounter.

BrakReal enemy materialization

map_node_enemies ma placeholdery, nie realne snapshoty armii przeciwnika.

BrakRewards playable flow

Rewards nie sa jeszcze w faktycznie przechodzonym flow gry.

BrakRandom Events

RandomEvent istnieje jako typ mapy, ale nie ma jeszcze gameplay flow.

Drugie do zrobienia

Predefined Armies Catalog

Cel: katalog authored armii dla PredefinedEnemyId, bossow i special encounterow. Pozniej ma zasilac map_node_enemies.army_snapshot_id.

Minimalne pola: predefinedArmyId, displayName, role, stacks[], a w stacku unitId, amount, formationSlot, skillIds[].

Runtime Snapshot -> Battle Scene Bridge

Cel: battle launch ma brac player army_snapshot_id i enemy army_snapshot_id, a maly adapter ma przygotowac format, ktory obecna scena walki potrafi przyjac.

RunMetagame launch record nie uzywa juz legacy-* handoff labeli. Do usuniecia zostaje fizyczny odczyt PlayerPrefs EnemyArmy / build-file w starej scenie walki.

Battle Map UI -> UI Shared Canvas

Do przerobienia po powrocie: UI battle map nie powinno siedziec w scenie battle. Ma byc przeniesione do UI Shared Canvas jak inne wspolne komponenty UI.
