[TARENA] PRD030_10 - Remove InMemory Production Usage Final Audit

Data: 2026-06-15. Raport dla manualnego sprawdzenia w Unity.

Werdykt

QA pass z wymaganym manualnym sprawdzeniem w Unity.

Produkcja Offline Mode ma teraz jeden punkt skladania DB-backed adapterow: OfflineModeDatabaseComposition.

Co powinno dzialac

  • Start Run, Run Map, Run Battle, Reward Map, Run Shop, Summary Value, Saved Armies i Battle Result powinny skladac swoje runtime serwisy przez OfflineModeDatabaseComposition.
  • Domyslne adaptery produkcyjne nie powinny tworzyc new InMemory...Store().
  • In-memory store'y zostaja tylko jako jawne test double dla EditMode testow.
  • UI controllery nie powinny wykonywac zapytan SQLite bezposrednio; DB dotyka store/repository/database infrastructure.
  • Domyslna baza TArenaOffline.db powinna byc otwierana/migrowana przed utworzeniem runtime DB-backed serwisow.

Jak ustawic scene

  1. Otworz scene albo prefab testowy z ekranami Offline Mode PRD019/PRD030.
  2. Upewnij sie, ze obiekty ekranow maja przypiete kontrolery: StartRunScreenController, PRD19_021_RunMapMockupController, RewardMapScreenController, RunShopScreenController, SummaryValueScreenController, SavedArmiesScreenController, BattleResultScreenController.
  3. Sprawdz, czy pola TMP, przyciski, karty nagrod/ofert i wiersze list sa przypiete tak jak w aktualnych prefabach. Nie trzeba dodawac legacy UnityEngine.UI.Text.
  4. Zacznij od ekranu Start Run. Po rozpoczeciu runu uzyj utworzonego, persystowanego run-* id jako wartosci runId na kolejnych ekranach.
  5. Dla Run Shop ustaw routeNodeId na aktywny shop node z Run Map. Dla Reward Map uzyj aktywnego reward/battle node z tego samego runu.
  6. Nie testuj tych ekranow na starym placeholderze offline-run, bo po tej zmianie produkcyjna sciezka oczekuje runu istniejacego w DB.

Jak przetestowac w Play Mode

  1. Wcisnij Play.
  2. Na Start Run wybierz armie i trase, rozpocznij run, a potem sprawdz brak bledow w Console.
  3. Przejdz na Run Map i wybierz dostepny node. Oczekiwane: stan mapy laduje sie z tego samego offline runu.
  4. Uruchom przygotowanie bitwy przez Run Battle. Oczekiwane: payload bitwy dostaje persystowane run_battle_id, a run_events ma wpis battle.
  5. Po symulowanym wyniku przejdz na Reward Map. Oczekiwane: wybor nagrody zapisuje choice, karty, snapshot przed/po oraz aktualizuje run gold.
  6. Wejdz w Run Shop, kup oferte i wyjdz. Oczekiwane: purchase zapisuje event, purchase detail, snapshot po zakupie i powrot do Run Map.
  7. Na Summary Value zapisz pre-final army do slota. Oczekiwane: poprzednia armia w slocie jest soft-deaktywowana, nowa zostaje aktywna.
  8. Na Saved Armies ustaw current defence. Oczekiwane: zapis i ponowne wejscie do ekranu pokazuja ten sam slot/defence z DB.
  9. Na Battle Result zapisz/przeladuj wynik. Oczekiwane: result wraca z domyslnej bazy Offline Mode, nie z osobnej preview DB.

Testy EditMode

Dodany plik: Assets/Scripts/Tests/EditMode/OfflineModeProductionCompositionTests.cs.

  1. Otworz Unity Test Runner: Window > General > Test Runner.
  2. Wejdz w zakladke EditMode.
  3. Uruchom OfflineModeProductionCompositionTests.
  4. Oczekiwane: 2 testy pass.

Te testy skanuja source runtime i nie wymagaja ustawionej sceny ani prefabow.
